Prompt Engineering — Il server costruisce un prompt strutturato con destinazione, durata, interessi e template di output. I dati semantici della destinazione (da DESTINATION_TAGS) arricchiscono il contesto.
Chiamata al LLM — Il server chiama llama-3.3-70b-versatile via Groq API con stream: true. Il modello genera ~1500 token in pochi secondi con temperatura 0.7.
Server-Sent Events (SSE) — Il server invia ogni chunk di testo come evento SSE (text/event-stream). Non è necessario attendere la risposta completa: il client la legge in tempo reale.
Rendering in Real-Time — Il client usa EventSource o ReadableStream per consumare gli eventi e aggiorna il DOM parola per parola, simulando l'effetto typewriter.
